# Support

## Where to ask

- **Questions / ideas** — [GitHub Discussions](https://github.com/kero168/slopguard/discussions)
- **Bugs** — [bug report issue](https://github.com/kero168/slopguard/issues/new?template=bug_report.yml)
- **A PR scored wrong** — [signal feedback issue](https://github.com/kero168/slopguard/issues/new?template=signal_feedback.yml).
  This is the most useful report you can file for this project.
- **Security problems** — see [SECURITY.md](./SECURITY.md); please do not open a public issue.

## What to include

- `slopguard --version`, Node version, and whether you used the CLI or the Action
- The command or workflow snippet
- If you can share it, the (redacted) PR JSON and diff — fixtures are how bugs
  get fixed and heuristics get tuned here

## Expectations

This is a volunteer-maintained project. Issues are triaged in a best-effort
manner; there is no SLA. Kind, complete reports get answered fastest.
